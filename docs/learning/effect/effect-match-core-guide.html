Effect Match 核心：型別縮窄與完整分支

Match 把「建立 matcher、加入分支、決定未命中語意」寫成可組合 pipeline。它的主要價值是 handler narrowing 與 discriminated union 的完整性檢查。

版本基準：effect@4.0.0-beta.98。｜Effect 學習路徑

三階段心智模型

const render = Match.type<PaymentState>().pipe(
  Match.tag("Pending", () => "Waiting"),
  Match.tag("Paid", ({ receipt }) => receipt),
  Match.tag("Failed", ({ message }) => message),
  Match.exhaustive,
)
  1. type 或 value 決定輸入。
  2. when／tag 增加分支並收窄 handler input。
  3. exhaustive、orElse 或 option-like completion 決定未命中語意。

為什麼不只用 switch

switch 對簡單 union 已經很好。Match 更適合需要可組合 patterns、nested structural matching 或以 pipeline 建立 reusable matcher 的地方。不要為了使用 library 把清楚的兩分支 switch 改得更抽象。

分支順序仍然重要

廣泛 pattern 放在前面可能吞掉精確分支。Match 提供型別協助，不會替你證明任意 predicate 互斥，也不會把 runtime object pattern 解讀成完整物件相等。

錯誤邊界

Tagged error union 特別適合 Match：每個 handler 取得精確 error type，exhaustive 又能在新增 error variant 時迫使 boundary 更新。

Pattern API 的選擇請讀Effect Match Pattern Reference。
