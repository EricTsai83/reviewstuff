Effect 的延遲執行

Effect 本身延遲執行，不代表用來建立 Effect 的 JavaScript 也會延遲。判斷 succeed、sync 與 suspend 時，要先問「哪一段工作應該到執行時才發生」。

版本基準：effect@4.0.0-beta.98。｜Effect 學習路徑

三個時間點

  1. JavaScript 建立 Effect description。
  2. Runtime 開始執行 description。
  3. Effect 內部產生成功、失敗或下一個 Effect。
API現在做什麼執行時做什麼
Effect.succeed(value)先計算並捕捉 value回傳既有 value
Effect.sync(() => value)保存 thunk呼叫 thunk 產生 value
Effect.suspend(() => effect)保存 thunk呼叫 thunk 取得下一個 Effect
const captured = Effect.succeed(Date.now())
const refreshed = Effect.sync(() => Date.now())
const deferredProgram = Effect.suspend(() =>
  enabled ? runFeature : Effect.void
)

每次執行都會重新呼叫

sync 與 suspend 不是一次性的 lazy cache。同一個 description 執行兩次，thunk 也會執行兩次。需要 memoization 時，必須使用明確的共享／快取機制。

Callback 可能已經提供延遲

flatMap、catch 或 andThen 接受的 callback 本來就在適當事件發生後才呼叫。不要因為看到副作用就機械式再包一層 suspend。

判斷規則

延遲普通值計算用 sync；延遲「選擇或建立下一個 Effect」用 suspend；已經存在的純值用 succeed。

遞迴、分支型別與其他用途請讀Effect.suspend 進階模式。
