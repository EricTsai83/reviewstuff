Effect Match Pattern Reference

先依資料形狀選 pattern，再決定 matcher 如何完成。這份參考集中 API 差異，不重複介紹 Match 的基礎心智模型。

版本基準：effect@4.0.0-beta.98。｜Effect 學習路徑

需求工具
literal、結構或 predicateMatch.when(pattern, handler)
discriminated union 的單一 _tagMatch.tag("Tag", handler)
同一 handler 接受多個 tagsMatch.typeTags 或對應 tag combinator
立即匹配一個 tagged valueMatch.valueTags(value, handlers)
任一 pattern 成立Match.whenOr(...patterns, handler)
所有 pattern 同時成立Match.whenAnd(...patterns, handler)

type 與 value

Match.type<A>() 建立可重用函式；Match.value(value) 立即處理既有值。若把過度收窄的 as const value 傳入，matcher 可能只看見單一 variant，失去 union exhaustiveness 的意義。

Completion

  • exhaustive：型別上必須完整處理。
  • orElse：提供 fallback。
  • Option-like completion：未命中本身是正常資料。

常見陷阱

  • 把廣泛 predicate 放在精確 tag 前。
  • 把 object pattern 當成 deep equality。
  • 認為自訂 predicate 自動證明 exhaustiveness。
  • handler 回傳彼此無關型別，卻沒有先定義 matcher 的共同輸出。
