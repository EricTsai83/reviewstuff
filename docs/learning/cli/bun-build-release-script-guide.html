Bun Build 與 Release Script

Build wrapper 的產品是 artifact，不是 application result。它應固定輸入、輸出與失敗條件，並讓本機與 CI 使用同一條可重現路徑。

版本基準：bun@1.3.14。Effect 是否適合 build script，取決於 script 是否真的需要共享 service 與 typed lifecycle。

可靠 wrapper 的責任

  1. 明確指定 entry point、target、output path 與 externalization policy。
  2. 在開始前驗證環境與必要檔案。
  3. 把 compiler diagnostics 保留為 build failure，不假裝成功。
  4. 完成後驗證 artifact 存在、可執行且版本資訊正確。
  5. Release 階段另處理簽章、notarization、checksum 與發佈。

何時使用 Effect

保留 Effect使用直接 Bun script
需要共享 FileSystem／process services、typed error mapping、scope 或可替換測試 Layer單一短流程，只呼叫 compiler、檢查結果並退出

不要只為風格一致把十行 script 包成完整 Layer graph；也不要在已有嚴格 process policy 時繞過它。選擇應由 failure、resource 與 testing requirements 決定。

Build、install 與 release 不同

  • Build：從 source 產生 artifact。
  • Install：把已驗證 artifact 放到使用者可執行的位置。
  • Release：簽署、產生 checksum、發佈並提供可追溯 metadata。
Production gate

不要把「compiler exit 0」當成完整成功。至少 smoke-test artifact、驗證 target architecture，並確保 release 上傳的是剛剛驗證過的同一份 bytes。
