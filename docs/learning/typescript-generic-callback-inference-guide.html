泛型與 Callback 推論

泛型函式的價值不只在接受很多型別，而是在型別層保留輸入與輸出的關係。過早寫成較寬的 return type，會讓這份關係消失。

固定型別和泛型關係不同

const identity = <T>(value: T): T => value

const widened = <T>(value: T): unknown => value

const a = identity("hello")
//    ^? "hello"

const b = widened("hello")
//    ^? unknown

unknown 沒有說謊，但它切斷了「輸出就是輸入型別」的契約。Return annotation 技術上正確，不代表它保存了有用資訊。

Callback 通常已經有 contextual typing

const lengths = ["a", "abcd"].map((value) => value.length)

map 已提供 parameter context，callback body 又能精確推論成 number。重複寫 (value: string): number 通常沒有增加邊界，只增加維護點。

何時仍值得標註 callback

  • Callback 本身被 export 或跨模組傳遞。
  • Library callback contract 很複雜，推論錯誤難以定位。
  • 你要刻意限制輸出，不讓 implementation detail 洩漏。
  • 需要防止未預期的 async return；這屬於另一個 void 問題。
