Effect.suspend 進階模式

核心用途是把「下一個 Effect 的建立」推遲到 runtime。遞迴是最重要的案例；動態策略與 TypeScript 分支統一則是次要工具。

版本基準：effect@4.0.0-beta.98。｜Effect 學習路徑

安全建立遞迴 Effect

const repeat = (remaining: number): Effect.Effect<void> =>
  remaining <= 0
    ? Effect.void
    : Effect.suspend(() =>
        task.pipe(Effect.andThen(repeat(remaining - 1)))
      )

沒有 suspend 時，repeat(remaining - 1) 可能在 description 建構階段一路遞迴。加上 suspend 後，每次 runtime 只建立下一層，讓 Effect runtime 管理執行與取消。

執行時選擇策略

const program = Effect.suspend(() =>
  currentMode() === "local" ? localProgram : remoteProgram
)

只有當策略必須在每次執行時重新觀察，才需要 suspend。若 mode 已是 Effect value，通常用 flatMap 更直接。

協助 TypeScript 統一分支

複雜泛型分支有時會在 eager expression 上得到不理想的 union。把整個 Effect-producing expression 放入 suspend，可以提供明確的共同 return boundary；但應先嘗試改善函式 contract，不要把 suspend 當成任意 type cast。

它不會處理什麼

  • 不自動捕捉 thrown exception；依來源使用 try／tryPromise 等 API。
  • 不 memoize。
  • 不保證更快。
  • 不取代 resource Scope。
測試

驗證建構時 thunk 尚未執行、每次 run 都重新執行、遞迴可被取消，並測試動態分支在兩次 run 之間改變時確實重新選擇。
