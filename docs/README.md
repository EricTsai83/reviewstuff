# Documentation

The documentation tree separates reusable teaching material from documents
whose truth is the current repository implementation.

| Area | Purpose |
| --- | --- |
| [`learning/`](./learning/README.md) | Independently readable tutorials, case studies, overviews, and learning paths |
| [`reference/`](./reference/reviewstuff-repository-structure.md) | Normative repository rules that must evolve with the codebase |
| [`system-design/`](./system-design/reviewstuff-git-pipeline-implementation-reference.html) | Current implementation references for specific subsystems |
| [`assets/`](./assets/learning-article.css) | Shared presentation assets used by learning articles |

## Document-role suffixes

- `*-learning-path`: ordered entry point for a topic series.
- `*-overview`: architecture map or navigation hub.
- `*-introduction`: conceptual starting point.
- `*-guide`: independently readable tutorial.
- `*-case-study`: one concrete application of broader concepts.
- `*-reference`: lookup-oriented material.
- `*-system-design`: reusable design reasoning rather than a live code snapshot.
- `*-implementation-reference`: code-coupled description of the current implementation.

Historical implementation plans remain under [`../plans/`](../plans/README.md)
and are not part of the teaching-document inventory.
