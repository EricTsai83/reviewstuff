ReviewReport Schema Version System Design

本文件定義 reviewstuff review --json 的版本策略。目標是保留必要的相容性資訊，同時避免在沒有獨立演進需求的巢狀資料上增加版本。

系統背景

review command 會產生一份 ReviewReport，terminal renderer 將它顯示給人閱讀，JSON renderer 則把它輸出給 script、CI 或其他工具。因為 JSON 可能由不同版本的 consumer 解析，所以完整 report 是一個需要版本的資料契約。

coverage 只存在於 ReviewReport 裡面，目前不會獨立儲存、傳輸或解析。它不是獨立契約，因此不需要自己的版本。

設計決策

只保留外層 ReviewReport.schemaVersion

目標格式使用 schemaVersion: 2。移除 coverage.schemaVersion，並把 ReviewCoverageV1 更名為 ReviewCoverage。

資料 是否版本化 原因
ReviewReport 是，版本 2 它是 --json 輸出的完整 machine-readable contract。
ReviewCoverage 否 它只由 report 包含，跟隨外層 report 一起演進。
Terminal text 否 它是給人閱讀的 presentation，不是 JSON schema。

目標資料契約

維持目前簡單的 TypeScript contract。系統只負責產生 report，現階段不需要額外導入 runtime schema 或 migration framework。

TypeScriptexport interface ReviewCoverage {
  readonly complete: boolean;
  readonly files: ReadonlyArray<ReviewFileCoverage>;
}

export interface ReviewReport {
  readonly schemaVersion: 2;
  readonly scope: ReviewScope;
  readonly summary: ReviewReportSummary;
  readonly coverage: ReviewCoverage;
  readonly findings: ReadonlyArray<Finding>;
}

runReview 必須明確回傳 Effect.Effect<ReviewReport, ...>，讓 TypeScript 在 report 的生產位置檢查版本與欄位。JSON renderer 保持單純：

TypeScriptexport const renderJsonReport = (report: ReviewReport): string =>
  JSON.stringify(report, undefined, 2);

版本演進規則

需要提升外層版本

  • 新增、移除或重新命名欄位。
  • 改變欄位型別或 enum 值。
  • 改變既有欄位的業務語意。

不需要提升版本

  • 內部實作重構。
  • terminal 顯示文字改變。
  • JSON indentation 或欄位輸出順序改變。
  • 修正 bug，使輸出重新符合既有契約。

這次變更使用 v2 或 v3

預設假設目前含有 coverage.schemaVersion 的 v2 尚未對外發布，因此直接整理成最終 v2。如果現有 consumer 已經依賴該欄位，則不得重新定義 v2，必須把外層版本提升為 v3。

實作範圍

  1. src/domain/review-file.ts
    將 ReviewCoverageV1 更名為 ReviewCoverage，移除其 schemaVersion。
  2. src/domain/report.ts
    改為引用 ReviewCoverage，外層維持 schemaVersion: 2。
  3. src/use-cases/run-review.ts
    移除建立 coverage 時的 schemaVersion: 1。
  4. 相關 tests
    更新 fixtures 與 JSON expectations，確認版本只出現在 report 最外層。

驗收標準

  • ✓ review --json 的最外層包含 schemaVersion: 2。
  • ✓ coverage 不包含 schemaVersion。
  • ✓ 專案中不再存在 ReviewCoverageV1。
  • ✓ bun run typecheck 通過。
  • ✓ report renderer、run-review 與 CLI e2e tests 通過。
