# To HTML Design System

Default visual system for standalone HTML articles produced by `to-html`. Keep this file compact: it defines article layout, core tokens, component styling, and checks. Code highlighting is generated by `scripts/highlight-code-blocks.mjs`; do not describe tokenizer internals here.

## Workflow

Apply this after the first complete draft and the `IMPROVE-HTML-ARTICLE.md` pass.

1. Preserve content, language, code examples, links, and semantic HTML.
2. Replace ad hoc styling with the tokens and CSS pattern below.
3. Do not add remote fonts, CDNs, external images, runtime syntax highlighters, frameworks, or runtime Mermaid.
4. After styling, run `bun .agents/skills/to-html/scripts/highlight-code-blocks.mjs <html-file>`.
5. Verify mobile and desktop readability.

## Core Tokens

Use only tokens the article needs. Add optional semantic colors only when a component uses them.

```css
:root {
  color-scheme: light;
  --clay: #D97757;
  --slate: #141413;
  --ivory: #FAF9F5;
  --oat: #E3DACC;
  --white: #FFFFFF;
  --gray-100: #F0EEE6;
  --gray-300: #D1CFC5;
  --gray-500: #87867F;
  --gray-700: #3D3D3A;
  --warning: #C78E3F;
  --space-sm: 8px;
  --space-md: 12px;
  --space-lg: 16px;
  --space-xl: 24px;
  --space-2xl: 32px;
  --space-3xl: 48px;
  --radius-sm: 4px;
  --radius-md: 8px;
  --content-width: 68ch;
}
```

## Base CSS

Use system fonts only. The whole article must not be a card.

```css
body {
  margin: 0;
  background: var(--ivory);
  color: var(--slate);
  font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
  font-size: 16px;
  font-weight: 430;
  line-height: 1.55;
  letter-spacing: 0;
}

main {
  max-width: var(--content-width);
  margin: 0 auto;
  padding: var(--space-3xl) var(--space-xl);
}

article {
  background: transparent;
  border: 0;
  border-radius: 0;
  box-shadow: none;
  padding: 0;
}

h1 {
  max-width: none;
  text-wrap: balance;
  overflow-wrap: normal;
}

p,
ul,
ol,
.deck,
.callout,
table,
pre {
  max-width: var(--content-width);
}

table,
pre {
  width: 100%;
}
```

## Components

- Links: `--clay`, underlined, visible underline offset.
- Inline code: `--gray-100` background, `--oat` border, `--radius-sm`, wraps safely on mobile.
- Code blocks: `--slate` background, `--ivory` text, `--gray-700` border, `--radius-md`, `--space-lg`, horizontal scroll.
- Tables: same width as prose, collapsed borders, `--gray-100` header, `--oat` borders, compact padding.
- Callouts: `--gray-100` background, `--oat` border, left accent using `--clay`; warning callouts may use `--warning`.
- Sources/reference sections: top border using `--oat`, smaller text.

## Code Highlight Colors

The bundled highlighter emits static token spans. Style those classes; do not add a runtime highlighter.

```css
.k { color: #f0a178; font-weight: 600; }
.s { color: #b7d28a; }
.n,
.p { color: #8fb6d8; }
.c { color: var(--gray-500); font-style: italic; }
.f { color: #e8c16f; }
.t { color: #f1d6a6; font-weight: 600; }
.o { color: #c8c3b7; }
```

## Mobile

```css
@media (max-width: 640px) {
  body { font-size: 15px; }
  main { padding: var(--space-xl); }
  pre code { font-size: 0.8125rem; line-height: 1.5; }
  p code,
  li code,
  td code { font-size: 0.9em; }
  table,
  .deck,
  .sources { font-size: 0.875rem; }
}
```

## Final Check

- No remote dependencies or runtime highlighters.
- Whole article is not wrapped in a visible card shell.
- Title has no awkward single-character orphan line.
- Text, callouts, tables, and code blocks align to the same content width.
- Code blocks preserve escaping and use static highlighting from the bundled script.
- Mobile layout has no unintended horizontal overflow except code/table scrolling.
